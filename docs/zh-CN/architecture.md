## 核心组件详解

### 1. 网络层 (Network Layer)

基于 Tokio 的异步网络处理：

```rust
// 简化的服务器启动代码
let listener = TcpListener::bind("0.0.0.0:6379").await?;

loop {
    let (socket, addr) = listener.accept().await?;
    tokio::spawn(handle_connection(socket, addr));
}
```

**特点：**
- 每个连接一个独立的 Tokio 任务
- 使用 `Framed` 进行 RESP 编解码
- 支持高并发连接

### 2. RESP 协议层

Redis Serialization Protocol 实现：

```
Simple Strings: +OK\r\n
Errors:         -ERR message\r\n
Integers:       :1000\r\n
Bulk Strings:   $6\r\nfoobar\r\n
Arrays:         *2\r\n$3\r\nfoo\r\n$3\r\nbar\r\n
```

**解析流程：**
1. 读取第一个字节判断类型
2. 根据类型读取后续数据
3. 解析 CRLF 分隔符
4. 递归解析嵌套数组

### 3. 命令层

命令处理架构：

```rust
// 命令分发
match command_name {
    "GET" | "SET" | "INCR" => string::handle(args),
    "HGET" | "HSET" => hash::handle(args),
    "LPUSH" | "RPOP" => list::handle(args),
    "SADD" | "SISMEMBER" => set::handle(args),
    "ZADD" | "ZRANGE" => zset::handle(args),
    _ => Err("Unknown command"),
}
```

### 4. 存储层

使用 DashMap 实现并发安全：

```rust
pub struct Storage {
    data: DashMap<String, TypedValue>,
}

pub enum TypedValue {
    String(String),
    Hash(HashMap<String, String>),
    List(VecDeque<String>),
    Set(HashSet<String>),
    ZSet(SkipList<String, f64>),
}
```

**为什么选择 DashMap？**
- 无锁并发读取
- 细粒度锁写入
- 性能优于 RwLock<HashMap>

### 5. 持久化层

#### AOF (Append-Only File)

```
┌─────────────────────────────────────────┐
│           AOF 写入流程                   │
├─────────────────────────────────────────┤
│                                         │
│  1. 命令执行 ──→ 2. 写入 AOF 缓冲区      │
│       │                    │            │
│       ↓                    ↓            │
│  3. 响应客户端 ←── 4. 根据策略刷盘        │
│       │                                 │
│       ↓                                 │
│  5. 后台重写 (AOF Rewrite)              │
│                                         │
└─────────────────────────────────────────┘
```

**同步策略：**
- `always`: 每个命令 fsync
- `everysec`: 每秒 fsync（默认）
- `no`: 由操作系统决定

#### RDB (Snapshot)

```
┌─────────────────────────────────────────┐
│           RDB 保存流程                   │
├─────────────────────────────────────────┤
│                                         │
│  1. fork() 子进程                        │
│       │                                 │
│       ↓                                 │
│  2. 子进程遍历数据库                      │
│       │                                 │
│       ↓                                 │
│  3. 压缩并写入 RDB 文件                   │
│       │                                 │
│       ↓                                 │
│  4. 原子替换旧文件                        │
│                                         │
└─────────────────────────────────────────┘
```

### 6. 复制层

主从复制架构：

```
┌──────────────┐                      ┌──────────────┐
│    Master    │                      │   Replica    │
│              │                      │              │
│  ┌────────┐  │     SYNC/PSYNC      │  ┌────────┐  │
│  │  Data  │  │◄───────────────────►│  │  Data  │  │
│  └────────┘  │                      │  └────────┘  │
│       │      │                      │       │      │
│       ↓      │     Command Prop    │       ↓      │
│  ┌────────┐  │────────────────────►│  ┌────────┐  │
│  │  AOF   │  │   (异步复制)         │  │ Apply  │  │
│  └────────┘  │                      │  └────────┘  │
└──────────────┘                      └──────────────┘
```

**复制流程：**
1. 从节点发送 `PSYNC` 命令
2. 主节点创建 RDB 快照
3. 主节点发送 RDB 到从节点
4. 主节点持续发送写命令

## 数据流

### 读操作流程

```
Client ──→ TCP ──→ RESP Decode ──→ Command Parse ──→ Storage Read ──→ Response
```

### 写操作流程

```
Client ──→ TCP ──→ RESP Decode ──→ Command Parse ──→ Storage Write ──→ AOF Append ──→ Response
```

## 性能优化

### 1. 零拷贝

- 使用 `Bytes` 类型避免内存复制
- RESP 解析器直接操作原始字节

### 2. 批处理

- 命令流水线支持
- 批量写入 AOF

### 3. 内存优化

- `smallvec` 减少小数组分配
- 字符串驻留（未来优化）

### 4. 锁优化

- DashMap 替代全局锁
- 命令级别的并发控制

## 故障处理

### 1. 连接断开

- 自动清理连接状态
- 未完成命令回滚

### 2. 持久化失败

- `stop-writes-on-bgsave-error` 配置
- 错误日志记录

### 3. 主从复制失败

- 自动重连机制
- 部分重同步 (PSYNC)

## 监控与调试

### 内置命令

- `INFO`: 服务器统计信息
- `CLIENT LIST`: 连接列表

### 日志级别

- `debug`: 详细调试信息
- `verbose`: 一般信息
- `notice`: 重要信息（默认）
- `warning`: 警告和错误
